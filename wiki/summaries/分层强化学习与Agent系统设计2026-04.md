# 分层强化学习与Agent系统设计（2026-04）

**来源**: 个人分析笔记
**类型**: 跨领域综合

## 核心洞察

2025-2026年爆发的顶级Agent系统（Claude Code、OpenClaw、Hermes Agent），其工程架构正在**像素级复刻**20年前分层强化学习（[[分层强化学习|HRL]]）的理论框架。

## 三大Agent系统与HRL的对应

### 1. Claude Code → Sub-agents = Options
- 主Agent遇到复杂任务时派生（Spawn）子Agent
- 子Agent在独立上下文窗口工作，完成后只汇报结果
- 主Agent思考深度从500步压缩到1步

### 2. OpenClaw → Tools = Primitive Actions
- 将LLM与触发器、内存、本地命令行/浏览器结合
- 像人类操作员一样长周期自主循环
- 被称为"套在大模型外面的装甲"

### 3. Hermes Agent → Skill Loop = Option Discovery
- 完成任务后复盘操作路径
- 将成功路径写成Markdown文件（Skill）
- 下次遇到类似任务直接调用Skill而非从零思考
- 运行自我创建技能的Agent速度提升40%

## 1:1 映射关系

| HRL 概念 | LLM Agent 实现 | 对应系统 |
|-----------|---------------|----------|
| 基础动作 (Primitive Actions) | 工具调用 (Tools/Function Calling) | 所有Agent |
| Options（选项/宏动作） | Skills / Sub-agents | Hermes / Claude Code |
| 状态空间爆炸 | 上下文窗口耗尽 | Claude Code（子代理隔离） |
| Option Discovery | 自我进化 / Skill Loop | Hermes Agent |

## 智能计算的唯一铁律

> 无论是MDP数学机器人，还是千亿参数LLM，只要面临序列决策的组合爆炸，唯一的解法就是**抽象（Abstraction）和分层（Hierarchy）**。
