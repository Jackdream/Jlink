# 关键要点

- **效率差距**: AI agent比普通工程师高效10-100倍，差距在架构不在智能
- **核心原则**: 薄harness，厚skills
- **Claude Code源码**: 51.2万行，秘密在harness层
- **Skill即方法调用**: 同一skill不同参数产生不同能力
- **Resolver路由**: 按需加载上下文，2万行CLAUDE.md精简到200行
- **潜在空间vs确定性**: 模型做判断，确定性工具执行
- **日志化**: 知识工作中AI超越数据库查询的关键

# 五个核心定义

## 1. Skill 文件

Skill是Markdown格式的过程文档，教模型"怎么做"，不是"做什么"。

### 关键特性
- 像一次方法调用，接受参数
- 同一过程，不同参数，产生截然不同能力
- 用模型本来就在思考的语言描述过程

### 示例
```
/investigate skill，7个步骤，3个参数：
TARGET、QUESTION、DATASET

→ 指向安全科学家 + 210万封邮件 → 医学研究分析师
→ 指向空壳公司 + FEC备案 → 法证调查员
```

## 2. Harness

运行LLM的程序，做四件事：
- 循环运行模型
- 读写文件
- 管理上下文
- 执行安全约束

### 薄harness原则
- 专用的、窄口径的工具
- Playwright CLI（100ms/操作） vs Chrome MCP（15秒/操作）
- 快75倍

### 反模式：厚harness薄skills
- 40多个工具定义吃掉一半上下文窗口
- 上帝工具，每次MCP往返2-5秒
- 每个REST端点都包成独立工具

## 3. Resolver

上下文的路由表，告诉模型"加载什么、什么时候加载"。

### 效果示例
- 开发者改prompt，有resolver → 模型先读`docs/EVALS.md`
- 没resolver → 直接发布
- 2万行CLAUDE.md → 200行（指向文档的指针）

## 4. 潜在空间 vs 确定性

| 类型 | 特点 | 适合任务 |
|------|------|----------|
| **潜在空间** | 智能所在，模型决策 | 判断、综合、模式识别 |
| **确定性** | 信任所在，相同输入相同输出 | SQL、编译、算术 |

### 常见错误
- 组合优化（确定性问题）强行塞进潜在空间
- 模型幻觉出看起来合理但错误的座位表

## 5. 日志化（Diarization）

让AI真正用于知识工作的关键步骤：
1. 模型读取某个主题的所有内容
2. 写出结构化档案
3. 从几十/几百份文档提炼单页判断

### 数据库做不到的
- 捕捉"说的"和"实际做的"的差距
- 需要同时读多个来源，同时在脑子里持有矛盾

# 三层架构

```
┌─────────────────────────────────────┐
│  厚 Skills（顶层）                   │
│  Markdown过程文档，编码判断/过程/知识  │
│  90%的价值在这里                     │
├─────────────────────────────────────┤
│  薄 CLI Harness（中间层）            │
│  ~200行代码，JSON进文本出            │
├─────────────────────────────────────┤
│  应用层（底层）                       │
│  QueryDB、ReadDoc、Search、Timeline │
│  确定性工具                          │
└─────────────────────────────────────┘
```

# 学习循环

```
检索 → 读取 → 日志化 → 计数 → 综合
  ↓
调查 → 日志化 → 重写Skill
```

### 实践案例
YC的6000名创始人匹配系统：
1. `/enrich-founder` skill拉取所有来源，日志化
2. `/match-breakout` 处理1200人，按行业聚类
3. `/match-lunch` 处理600人，跨行业随机匹配
4. `/match-live` 处理在场馆的人，200ms配对
5. `/improve` skill读取NPS调查，重写规则

结果：12%→4%的"还行"评分，skill自动学会

# 核心指令

> 你不允许做一次性的工作。如果我让你做某件事，而这件事属于将来还会发生的那种，你必须：
> 1. 第一次手动做，处理3-10个样本
> 2. 给我看输出
> 3. 如果我批准，把它写成一个skill文件
> 4. 如果它应该自动运行，放进cron
>
> 测试标准：如果我需要问你同一件事两次，你就失败了。

# 解读/意义

1. **架构优先于模型**: 10-100倍差距来自架构，不是模型本身
2. **Skills是永久升级**: 每次模型改进，所有skill自动变好
3. **系统复利增长**: 构建一次，永远运行
4. **知识工作革命**: 日志化让AI能做数据库做不到的事
