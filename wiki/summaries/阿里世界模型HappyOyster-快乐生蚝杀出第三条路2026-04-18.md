## 核心事件

**2026年4月**，阿里发布 **HappyOyster（快乐生蚝）** —— 可实时构建和交互的开放式世界模型产品。

> 名字借用莎翁名言 "The world is your oyster."（世界是你的牡蛎）

**所属团队**：Alibaba Token Hub（ATH）创新事业群（2026年3月成立）

**前作**：Happy（欢乐马）—— 曾登顶 Artificial Analysis 排行榜

## 产品定位

| 对比维度 | 文生视频模型（Sora/可灵） | HappyOyster（世界模型） |
|----------|-------------------------|------------------------|
| **交互模式** | 一次性生成，封闭流程 | 实时交互，开放式 |
| **用户控制** | 输入→等待→收成片 | 随时介入、改变、重写 |
| **时间维度** | 预设终点 | 无预设终点，持续演化 |
| **响应方式** | 无中间干预空间 | 任意节点实时响应 |

## 两大核心功能

### 1. 漫游（Wander）

**定位**：首个支持任意风格、无限互动的通用世界模型

| 特性 | 说明 |
|------|------|
| 输入方式 | 文本或图像 |
| 控制维度 | 角色（Character）+ 场景（Scene）分开设定 |
| 视角切换 | 第一人称 / 第三人称 |
| 实时控制 | WASD 控制角色移动，方向键控制镜头 |
| 生成时长 | 1 分钟以上实时位移和镜头控制 |
| 音画同步 | 自动生成契合场景的 BGM |

**实测案例**：
- 80年代巴黎街头（金发女模特）
- 吉卜力风格骑行场景
- 梵高画作世界漫游

### 2. 导演（Direct）

**定位**：基于世界模型的实时 AI 视频导演引擎

| 特性 | 说明 |
|------|------|
| 生成时长 | 连续生成 3 分钟 720p 实时视频 |
| 实时操控 | 文字指令控制镜头、调度角色、改变剧情 |
| 节点编辑 | 在视频任意节点实时改变内容 |
| 无缝衔接 | 变化与故事情节自然衔接，无需重新渲染 |

**实测案例**：
- 吉卜力风格小女孩撑伞场景
- 实时追加：小猫跑来 → 女孩蹲下抚摸

## 技术架构

### 核心挑战与解决方案

| 挑战 | 解决方案 |
|------|----------|
| **实时响应速度** | 流式生成框架，高维视频压缩为动态 latent state |
| **长时序一致性** | 持续状态复用机制，历史注意力状态连续传递 |
| **音画协同** | 统一音视频生成框架，同步生成视觉与听觉信号 |
| **在线注入控制** | 文本/图像/指令作为可在线注入的条件变量 |

### 技术路径对比

| 方向 | 代表 | 路线特点 |
|------|------|----------|
| **谷歌向左** | Google Genie | 实时交互式世界建模，多模态统一表达和音视频联合生成有局限 |
| **李飞飞往右** | World Labs | 3D 空间结构化重建，侧重几何一致性而非像素空间长时序动态 |
| **阿里第三条路** | HappyOyster | 像素空间长时序、实时可交互动态模拟 + 音视频联合生成 |

## 局限与挑战

| 挑战 | 说明 |
|------|------|
| 长时序物理一致性 | 生成时间越长，物理规律约束越难维持 |
| 复杂场景因果推理 | 多对象交互的因果关系建模 |
| 现实世界规律理解 | 对真实世界物理规律的深度建模 |

> "世界模型整体仍处于早期阶段。HappyOyster 是目前最接近『可用产品』形态的探索之一，但探索就意味着边界尚未确定。"
