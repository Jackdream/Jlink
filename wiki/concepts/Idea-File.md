# Idea File

## 定义

[[Karpathy]] 提出的原始概念，是 **LLM Wiki** 的抽象原型。

## 核心思想

**想法文件（Idea File）** 是一个由 LLM 维护的、结构化的个人知识库：

- 将零散的想法、阅读笔记、灵感整理成结构化文档
- LLM 负责维护交叉引用、更新摘要、标注矛盾
- 人类专注于输入和审阅，LLM 负责繁琐的 bookkeeping

## 与 LLM Wiki 的关系

| Idea File | LLM Wiki |
|-----------|----------|
| 抽象概念 | 具体实现 |
| 个人想法管理 | 系统化知识库 |
| 原型验证 | 生产级工作流 |

## 关键特征

1. **增量构建** - 新想法不断加入，自动整合到已有结构
2. **交叉引用** - 想法之间自动建立链接
3. **矛盾标注** - 发现不一致时主动标记
4. **综合更新** - 新信息触发相关页面的级联更新

## 实践转化

[[Zen]] 将 Idea File 概念转化为可运行的 **LLM Wiki** 系统：

- 使用 [[Hermes]] 作为执行引擎
- 基于 [[Obsidian]] 作为存储层
- 建立 `ingest/query/lint/status` 四个核心命令
