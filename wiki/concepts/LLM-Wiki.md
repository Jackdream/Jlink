# LLM Wiki

## 定义

由 [[Karpathy]] 提出的知识管理理念：**让 LLM 增量构建一个持久化的知识库**，而非每次查询时临时检索。

## 核心原则

1. **知识是编译的，不是重新推导的** - 在摄入时完成工作
2. **Wiki 是持久化的** - 交叉引用已存在，矛盾已标记，综合已完成
3. **LLM 负责维护** - 用户负责来源、探索、提问；LLM 负责其他一切

## 与 RAG 的对比

| 维度 | RAG | LLM Wiki |
|------|-----|----------|
| **查询方式** | 向量检索 + 生成 | 直接读取已编译页面 |
| **知识状态** | 原始文档，无预处理 | 结构化、交叉引用、已综合 |
| **维护成本** | 低（只存原始文档） | 接近零（LLM 自动维护） |
| **响应质量** | 依赖检索精度 | 依赖 Wiki 编译质量 |

## 三层结构

```
llm_wiki/
├── raw/              # 原始来源（不可变）
├── wiki/             # 编译后的知识页面
│   ├── entities/     # 实体页面
│   ├── concepts/     # 概念页面
│   ├── summaries/    # 来源摘要
│   ├── analyses/     # 分析页面
│   └── synthesis/    # 综合页面
├── index.md          # 内容索引
├── log.md            # 操作日志
└── CLAUDE.md         # Schema 约定
```

## 四个核心操作

| 操作 | 含义 | 状态 |
|------|------|------|
| **Ingest** | 摄入源文件，LLM 更新所有相关页面 | 核心操作 |
| **Query** | 基于 wiki 回答问题 | 待完善 |
| **Lint** | 体检，检查矛盾、孤儿页面、缺失链接 | 待完善 |
| **Status** | 查看 wiki 状态 | 基础功能 |
