# HappyOyster（快乐生蚝）

## 定义

[[阿里巴巴]] Alibaba Token Hub（ATH）创新事业群于 2026 年 4 月发布的**开放式世界模型产品**。

> 名字源自莎翁名言 "The world is your oyster."（世界是你的牡蛎）

## 核心定位

可实时构建和交互的开放式世界模型，区别于文生视频模型的一次性生成流程。

## 两大功能

### 1. 漫游（Wander）

| 特性 | 说明 |
|------|------|
| 定位 | 首个支持任意风格、无限互动的通用世界模型 |
| 输入 | 文本或图像 |
| 控制 | 角色 + 场景分开设定，第一/第三人称切换 |
| 交互 | WASD 移动，方向键控制镜头 |
| 时长 | 1 分钟以上实时控制 |
| 音效 | 自动生成契合场景的 BGM |

### 2. 导演（Direct）

| 特性 | 说明 |
|------|------|
| 定位 | 实时 AI 视频导演引擎 |
| 时长 | 连续生成 3 分钟 720p 视频 |
| 操控 | 文字指令实时控制镜头、角色、剧情 |
| 节点编辑 | 任意节点实时改变内容 |
| 响应 | 无需重新渲染，即时响应 |

## 技术架构

| 技术 | 作用 |
|------|------|
| 流式生成框架 | 高维视频压缩为动态 latent state，降低计算开销 |
| 持续状态复用 | 历史注意力状态连续传递，维持长时序一致性 |
| 统一音视频生成 | 视觉与听觉信号同步生成，跨模态时间对齐 |
| 在线条件注入 | 文本/图像/指令作为可实时注入的条件变量 |

## 技术路线对比

| 方向 | 代表 | 特点 |
|------|------|------|
| 谷歌向左 | Google Genie | 实时交互，多模态统一表达有局限 |
| 李飞飞往右 | World Labs | 3D空间重建，侧重几何一致性 |
| **阿里第三条路** | **HappyOyster** | **像素空间长时序动态模拟 + 音视频联合生成** |
