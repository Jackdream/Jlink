# 分层强化学习 (Hierarchical Reinforcement Learning)

## 定义

分层强化学习是一种将复杂强化学习问题分解为多个层级子问题的学习方法，通过上层控制器和下层控制器的协作来提升学习效率。

## 核心思想

```
┌─────────────────────────────────────────────────────┐
│                    上层控制器                        │
│   (长时间跨度) → 选择 Option/Goal → 提供内在奖励    │
└─────────────────────────────────────────────────────┘
                        ↓ 传递
┌─────────────────────────────────────────────────────┐
│                    下层控制器                        │
│   (短时间跨度) → 根据 Option/Goal → 选择 Action     │
└─────────────────────────────────────────────────────┘
```

## 解决的问题

### 1. 稀疏奖励问题

- 当环境奖励过于稀疏时，智能体长期无法获得正奖励样本
- 通过分层，上层控制器传递给下层控制器**内在奖励（intrinsic reward）**
- 即使外部奖励为0，下层控制器依然能获得奖励

### 2. 样本效率问题

- 提升**样本利用效率（Sample Efficiency）**
- 分解复杂任务，降低学习难度

## 两大类别

| 类别 | Option | Goal |
|------|--------|------|
| 定义 | 子策略 | 目标 |
| 层面 | 策略层面 | 目标层面 |
| 代表算法 | Option-Critic | UVFA, HER, H-DQN, FeUdal, HIRO, HAC |
| 主要应用 | 解决复杂任务 | 解决稀疏奖励 |

## Option vs Goal 的区别

> **Option 更像是解决复杂任务**：把复杂策略分解为简单子策略

> **Goal 则偏重解决稀疏奖励问题**：通过内部激励让智能体在稀疏环境下学习

## 为什么"冷门"？

1. **Option/Goal自动学习的困难**：大多数论文认为人为设计困难，但让智能体自己学反而增加复杂性
2. **理论与实践的Gap**：学术界方法与工业界实际需求存在错配
3. **通信领域的困境**：实际场景往往有丰富先验知识，不需要智能体自己找

## LLM Agent 中的映射（1:1 对应）

2025-2026年顶级Agent系统正在复刻HRL理论框架：

| HRL 概念 | Agent 实现 | 案例 |
|----------|-----------|------|
| 基础动作 | 工具调用 (Bash/Read/Write) | OpenClaw |
| Options（宏动作） | Skills / Sub-agents | Claude Code 子代理、Hermes Skill |
| 状态空间爆炸 | 上下文窗口耗尽 | 子代理独立上下文隔离 |
| Option Discovery | 自我进化 / Skill Loop | Hermes Agent 自动创建 Skill |

### 关键机制

- **子代理 (Sub-agent)**: Claude Code 派生子Agent在独立上下文工作，完工后只汇报结果 — 思考深度从500步压缩到1步
- **技能缓存 (Skill Loop)**: Hermes Agent 执行→评估→提取成功路径→写为 Skill 文件保存，下次直接调用，速度提升40%

核心铁律：序列决策的组合爆炸，唯一解法就是**抽象+分层**。
