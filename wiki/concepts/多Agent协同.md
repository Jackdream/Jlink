# 多Agent协同

多个 AI Agent 协作完成任务的技术范式。

## 核心概念

多 Agent 协同是指：
- 多个独立的 Agent 共享上下文
- 动态创建子 Agent
- 任务委托与分发
- 团队协作模式

## 与单 Agent 的区别

| 维度 | 单 Agent | 多 Agent 协同 |
|------|----------|--------------|
| 任务处理 | 串行执行 | 并行/分层执行 |
| 专业化 | 通用 | 可专业化分工 |
| 复杂度 | 适合简单任务 | 适合复杂任务 |
| 协调成本 | 无 | 需要协调机制 |

## OpenHarness 的多 Agent 能力

| 能力 | 说明 |
|------|------|
| 子 Agent 创建 | 动态创建专门负责某个领域的 Agent |
| 任务委托 | 将复杂任务拆解分配给不同 Agent |
| 团队注册 | 多个 Agent 形成团队协同工作 |
| 后台任务管理 | 支持后台运行和状态追踪 |

## Anthropic 的"角色分离"实践

一个 Agent 负责**写代码**，另一个 Agent 负责**评审**：
- 两者独立运行
- 避免"自己给自己打分"的盲区
- 提高输出质量

## 应用场景

- 复杂项目开发（分工明确）
- 代码审查与重构
- 多领域知识整合
- 团队协作模拟
