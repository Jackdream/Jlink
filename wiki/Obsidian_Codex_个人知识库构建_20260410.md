# Obsidian + Codex 个人知识库构建

## 核心思路

基于 Andrej Karpathy 的 LLM Wiki 思路，利用 Obsidian + Codex 构建个人知识库。工作量集中在反复迭代扩充知识，技术难度很低。

## 构建过程

- 4月3日开始，5天时间完成
- 迁移了2018年以来的所有个人积累内容（本地文件 + 有道云笔记）

## 解决的问题

解决了以下痛点：
- 依靠自己记忆和理解查找素材效率低
- 文件检索能力有限
- 网页对话限制上传附件体量
- 在复杂目录中大海捞针式搜索

## 知识库规模

- **3800+** 笔记总量
- **2000+** 原始笔记文件和备份素材
- **800+** 正式知识条目
- **3条** 主线：论文研究线、跨域知识整理线、工作项目线

## 目录结构

```
Inbox
├── Projects
├── Areas
├── Resources
├── Archive
├── AI Workspace
└── Worklog
```

> Worklog 是额外添加的，用于多智能体并行工作时记录 todo 和 worklog

## 技术优势

| 传统知识库 | LLM Wiki 方案 |
|-----------|--------------|
| 文件目录分类 | 双向链接连接 |
| 树形管理结构 | 网状知识结构 |
| 圈地让大模型扫 | 白盒 + 本地文件 + 云同步 |
| 大模型专用 | 人和 AI 都能用 |

## 核心价值

- 双向链接打通不同枝干的笔记
- 连接不同分支方向的知识
- 通过 Obsidian Cli 和智能体访问
- 挪开了迁移成本和构建双向链接的成本
